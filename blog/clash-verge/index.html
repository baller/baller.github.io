Linux 服务器使用 Clash Verge 教程

这是一篇从旧博客迁移来的服务器环境笔记，记录如何在 Linux 上通过命令行安装 Clash、配置订阅文件、设置 systemd 服务以及使用代理环境变量。

安装与配置

下载对应架构的 Clash 程序后，将可执行文件放到合适目录，并赋予执行权限。首次启动时需要 Country.mmdb 文件，也可以手动下载后放入配置目录。

sudo chmod +x /usr/local/bin/clash
curl -o ~/.config/clash/config.yaml 'subscription-url'

作为 systemd 服务运行

通过 systemd 可以让 Clash 常驻后台并支持开机启动。服务配置中需要指定 Clash 可执行文件和配置目录。

[Unit]
Description=Clash daemon
After=network.target

[Service]
Type=simple
Restart=always
ExecStart=/usr/local/bin/clash -d /home/xxx/.config/clash

[Install]
WantedBy=multi-user.target

使用代理

服务启动后，可以通过环境变量让命令行工具走本地代理端口。

export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7890
